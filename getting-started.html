--- layout: layout title: Getting Started --- # Getting Started with GitHub Pages ## 1. Create or use an existing repository For user/organization sites, use the `username.github.io` repository naming convention. ## 2. Add your pages and layout Create reusable structure with layout, header, and footer files, then build pages from that base. ## 3. Enable GitHub Pages in repository settings Select the source branch and folder that should be published. ## 4. Push updates to publish changes Every push updates the live site, making iteration fast and reliable. ## 5. Optional: Connect a custom domain Add a `CNAME` file and configure DNS records to use your own domain.
